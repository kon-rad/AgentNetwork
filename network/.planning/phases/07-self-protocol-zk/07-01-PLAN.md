<task type="auto">
  <name>Task 2: Create POST /api/self/verify endpoint</name>
  <files>src/app/api/self/verify/route.ts</files>
  <action>
    Create `src/app/api/self/verify/route.ts` with a POST handler that:

    1. Parses JSON body expecting: `{ attestationId, proof, publicSignals, userContextData }`
    2. Validates all four fields are present -- if missing, return JSON `{ status: 'error', result: false, reason: 'Missing required fields' }`
    3. Calls `verifySelfProof(attestationId, proof, publicSignals, userContextData)` from `@/lib/chain/self`
    4. Checks `result.isValidDetails.isValid`:
       - If false: return `{ status: 'error', result: false, reason: 'Verification failed' }`
       - If true: extract `result.userData?.userIdentifier` as wallet address
    5. On valid proof with wallet address:
       - `getDb().prepare('UPDATE agents SET self_verified = 1 WHERE wallet_address = ?').run(userWallet)`
    6. Return `{ status: 'success', result: true }`
    7. Wrap in try/catch -- on error return `{ status: 'error', result: false, reason: error.message }`

    IMPORTANT: Self Protocol convention returns HTTP 200 for ALL responses (success and failure). The status is in the JSON body, not the HTTP status code. Do NOT return 4xx/5xx for verification failures.

    Import getDb from '@/lib/db'. Follow the pattern of other API routes in the project (e.g., src/app/api/chain/upload/route.ts).
  </action>

  </verify>
