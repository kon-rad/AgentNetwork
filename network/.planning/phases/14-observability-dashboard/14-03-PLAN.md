Create `src/app/agent/[id]/observe/page.tsx` — a cyberpunk-styled observability dashboard.

**"use client" directive** — this is a client component (Supabase Realtime requires browser WebSocket).

**State:**
- `events: AgentEvent[]` — Realtime event feed (newest first — prepend on insert)
- `accessDenied: boolean` — shows access denied state on 401/403
- `loading: boolean`
- `agentName: string`
- `activeTab: 'events' | 'files'`
- `files: FileEntry[]` — from /api/agents/[id]/files
- `filesLoading: boolean`
- `expandedEvents: Set<string>` — which tool_call/llm_call event ids are expanded

**FileEntry type (inline in this file — NOT added to types.ts):**
```typescript
interface FileEntry {
  name: string;
  path: string;
  type: 'file' | 'dir';
  size?: number;
  modified?: string;
}
```

**Ownership check on mount:**
Reuse the same pattern as the chat page — fetch `/api/agents/${id}/chat` (GET, which already calls requireOwnership). If response is 401 or 403, set `accessDenied=true`. This avoids adding a dedicated auth check endpoint.

**Load recent events on mount:**
Fetch the 50 most recent events from Supabase using the browser client:
```typescript
const supabase = createClient(); // from @/lib/supabase/browser
const { data } = await supabase
  .from('agent_events')
  .select('*')
  .eq('agent_id', id)
  .order('created_at', { ascending: false })
  .limit(50);
if (data) setEvents(data as AgentEvent[]);
```
Note: This uses the anon key (browser client) — RLS on agent_events is not enforced in v2.0 (same as chat_messages). The ownership check above is application-layer enforcement.

**Supabase Realtime subscription (OBS-04):**
```typescript
const channel = supabase
  .channel(`agent-events-${id}`)
  .on(
    'postgres_changes',
    {
      event: 'INSERT',
      schema: 'public',
      table: 'agent_events',
      filter: `agent_id=eq.${id}`,
    },
    (payload) => {
      setEvents((prev) => [payload.new as AgentEvent, ...prev]);
    }
  )
  .subscribe();
// cleanup:
return () => { supabase.removeChannel(channel); };
```

**Tab: Events feed (OBS-01, OBS-02, OBS-03)**

Render a scrollable list of event rows. Each row shows:
- Event type badge (color-coded): `llm_call`=cyan, `tool_call`=purple, `turn_start`=green, `turn_complete`=green, `error`=red
- Relative timestamp (e.g. "2s ago" — use a simple helper: `formatRelTime(created_at)`)
- Brief summary text based on event_type:
  - `llm_call`: `{model} — {input_tokens} in / {output_tokens} out tokens` (cast `payload as { model: string, input_tokens: number, output_tokens: number }`)
  - `tool_call`: `{tool_name} ({duration_ms}ms)` (cast `payload as { tool_name: string, duration_ms?: number }`)
  - `turn_start`: `Turn started`
  - `turn_complete`: `Turn complete`
  - `error`: `Error: {message}` (cast `payload as { message: string }`)
- For `tool_call` and `llm_call` rows: a clickable chevron icon that toggles expansion into a detail panel showing:
  - `tool_call`: tool_name, input (JSON pre-formatted), output (JSON pre-formatted), duration_ms
  - `llm_call`: model, input_tokens, output_tokens (renders as the "Token Usage" panel — OBS-02)

**Token usage summary panel (OBS-02):**
Above the event list, show cumulative token totals computed from all `llm_call` events in state:
```
Total tokens: {sum(input_tokens + output_tokens)} | Input: {sum(input_tokens)} | Output: {sum(output_tokens)}
```
Use a `useMemo` that reduces over events filtered to `event_type === 'llm_call'`.

- Directory rows: folder icon + bold name

- Empty state: "No workspace files yet — agent hasn't run a task."

**Access denied state:** Same pattern as chat page — show a red "Access denied" card with "This dashboard is only visible to the agent's owner."

**Navbar link:** Do NOT modify the Navbar in this plan — the link to /observe can be added to the agent profile page separately. This page is self-contained.
