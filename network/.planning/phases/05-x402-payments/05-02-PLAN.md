  <files>

  <action>
    Replace the existing bounty complete route with USDC payment integration:

    1. Import transferUsdc from "@/lib/chain/usdc" (created in plan 01)
    2. Keep existing validation (bounty exists, status is "claimed")
    3. Look up the claiming agent's wallet_address from agents table using bounty.claimed_by
    4. If agent not found, return 404
    5. Set bounty status to "pending_payment" before attempting transfer
    6. Call transferUsdc(agent.wallet_address, bounty.reward_amount || "0")
    7. On success: update bounty with status="completed", tx_hash=txHash, deliverable_url, completed_at
    8. On failure: set status="payment_failed", return 502 with error message (use 502 per project convention for upstream service failures)
    9. Return JSON with tx_hash and status

    The existing route accepts { deliverable_url, tx_hash } in body. Change to only accept { deliverable_url } — tx_hash is now generated server-side from the actual USDC transfer.

    Handle edge case: if reward_amount is null or "0", skip the USDC transfer and complete without payment (some bounties may not have monetary rewards).
  </action>
  <verify>
    - File compiles: `npx tsc --noEmit src/app/api/bounties/[id]/complete/route.ts`
    - Route imports transferUsdc from "@/lib/chain/usdc"

    - Route catches transfer errors and sets "payment_failed" status
  </verify>
  <done>
