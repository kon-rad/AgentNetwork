Create src/app/subscribe/[agentId]/page.tsx as a "use client" component.

**State machine — 6 states:**
```typescript
type PaymentState =
  | 'idle'        // initial: show Subscribe button
  | 'prompting'   // wallet modal open / awaiting signature
  | 'pending'     // tx submitted, awaiting confirmation (show tx hash + BaseScan link)
  | 'confirming'  // tx confirmed on-chain, calling POST /api/subscriptions
  | 'launching'   // subscription recorded, NanoClaw notified
  | 'error'       // any failure — show message + retry
```

**wagmi hooks to use (existing stack, wagmi v2.x — do NOT use v3 APIs):**
```typescript
import { useWriteContract, useWaitForTransactionReceipt } from 'wagmi'
import { erc20Abi, parseUnits } from 'viem'
import { useAccount } from 'wagmi'

const USDC_CONTRACT = '0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913' as const
const REQUIRED_AMOUNT = parseUnits('100', 6) // 100_000_000n
```

**Subscribe button handler:**
```typescript
const { writeContract, data: txHash, isPending, isError, error } = useWriteContract()

function handleSubscribe() {
  setState('prompting')
  writeContract({
    address: USDC_CONTRACT,
    abi: erc20Abi,
    functionName: 'transfer',
    args: [process.env.NEXT_PUBLIC_TREASURY_ADDRESS as `0x${string}`, REQUIRED_AMOUNT],
  })
}
```

Note: TREASURY_ADDRESS must be exposed as `NEXT_PUBLIC_TREASURY_ADDRESS` for client-side use (the env var used in the API route is server-side `TREASURY_ADDRESS`; the client needs the public variant for the wagmi call).

**Transaction receipt watcher:**
```typescript
const { isLoading: isConfirming, isSuccess: isConfirmed } = useWaitForTransactionReceipt({
  hash: txHash,
})
```

**useEffect to drive state transitions:**
- txHash appears → set state to 'pending'
- isConfirmed → set state to 'confirming', then call POST /api/subscriptions with `{ tx_hash: txHash, agent_id }`
- POST returns 201 → set state to 'launching'
- Any error → set state to 'error', store error message

**UI per state:**
- `idle`: Show agent display_name, "100 USDC / month", blue cyber Subscribe button. If wallet not connected, show "Connect wallet to subscribe" (use `useAccount().isConnected`). If existing active subscription, show "Renew Subscription" button instead.
- `prompting`: Show "Waiting for wallet approval..." with spinner
- `pending`: Show "Transaction pending" + clickable tx hash link to `https://basescan.org/tx/${txHash}` + spinner
- `confirming`: Show "Confirming on-chain..." with spinner
- `launching`: Show "Agent Launching!" + green checkmark + link back to `/agent/${agentId}`
- `error`: Show error message in red + "Try again" button that resets to `idle`

**Check existing subscription on mount:** Fetch `GET /api/subscriptions/${agentId}` on load. If `has_active: true`, set `hasActiveSubscription = true` and display expiry date.

**Fetch agent data on mount:** GET `/api/agents/${agentId}` to show agent display_name.

Style using existing Tailwind + cyberpunk design tokens (glass-card, border-cyber-cyan, text-cyber-cyan, etc.) matching the established aesthetic in other pages.
