<task type="auto">
  <name>Task 1: Create agent registration API route</name>
  <files>src/app/api/agents/[id]/register/route.ts</files>
  <action>
Create POST /api/agents/[id]/register that orchestrates the full ERC-8004 registration flow:

1. **Extract agent ID** using Next.js 16 async params pattern: `const { id } = await params`
2. **Load agent from DB** — return 404 if not found
3. **Idempotency check** — if `agent.erc8004_token_id` is already set, return 200 with existing registration info (token ID, basescan URL). Do NOT re-register.
4. **Generate agent.json** — call `buildAgentCard(agent)` from `@/lib/agent-card`
5. **Upload agent.json to Filecoin** — call `uploadToFilecoin(agentCard, \`agent_card_${agent.id}.json\`)` from `@/lib/chain/filecoin` directly (server-side, no HTTP round-trip). Also persist to filecoin_uploads table using getDb().
6. **Register on-chain** — call `registerAgent(retrievalUrl)` from `@/lib/chain/erc8004` where retrievalUrl is the Filecoin CDN URL (e.g., `https://cdn.filecoin.cloud/{pieceCid}`)
7. **Store token ID in DB** — `UPDATE agents SET erc8004_token_id = ?, updated_at = datetime('now') WHERE id = ?` using agentId.toString()
8. **Generate and upload agent_log.json** — call `buildAgentLog(agent)`, `addLogEntry(log, { action: 'register_identity', status: 'success', details: { txHash, agentId: agentId.toString(), agentURI: retrievalUrl } })`, then upload to Filecoin with `uploadToFilecoin(log, \`agent_log_${agent.id}.json\`)`. Persist to filecoin_uploads as type 'agent_log'.
9. **Return 201** with `{ agentId: agentId.toString(), txHash, basescanUrl: \`https://sepolia.basescan.org/token/0x8004A818BFB912233c491871b3d84c89A494BD9e?a=\${agentId}\`, filecoinUrl: retrievalUrl }`

Error handling:
- Wrap on-chain calls in try/catch. If registerAgent fails, return 502 with error details (matches filecoin upload route pattern).
- If Filecoin upload fails before registration, return 502.
- Unexpected errors return 500.

Import `uploadToFilecoin` from `@/lib/chain/filecoin` directly — do NOT use internal HTTP fetch to /api/chain/upload.

For the filecoin_uploads DB insert, use the same pattern as the upload route: `crypto.randomUUID()` for ID, insert agent_id, upload_type, piece_cid, retrieval_url, name.

  <action>
Create POST /api/agents/[id]/feedback for submitting ERC-8004 reputation feedback:

1. **Extract agent ID** using Next.js 16 async params pattern
2. **Load agent from DB** — return 404 if not found
3. **Validate registration** — if `agent.erc8004_token_id` is null, return 400 with error "Agent is not registered on ERC-8004"
4. **Parse request body** — expect `{ value: number, tag1?: string, tag2?: string }`. Validate value is a number between 1-10. Default tag1 to "quality", tag2 to agent.service_type or "general".
5. **Submit feedback** — call `submitFeedback(BigInt(agent.erc8004_token_id), value, tag1, tag2)` from `@/lib/chain/erc8004`
6. **Return 201** with `{ txHash, agentId: agent.erc8004_token_id, basescanUrl: \`https://sepolia.basescan.org/tx/\${txHash}\` }`

Also add a GET handler that reads the agent's reputation summary:
1. Load agent, validate erc8004_token_id exists
2. Call `getReputationSummary(BigInt(agent.erc8004_token_id))` from `@/lib/chain/erc8004`
3. Return `{ count: count.toString(), value: value.toString(), decimals }` (convert bigints to strings for JSON)

Error handling: 502 for on-chain failures, 500 for unexpected.
