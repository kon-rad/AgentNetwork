<task type="auto">
  <name>Task 2: Create agent action functions with logging and idempotency</name>
  <files>src/lib/autonomous/agent-actions.ts</files>
  <action>
Create `src/lib/autonomous/agent-actions.ts` with these exported async functions. Each takes an Agent and AgentLog, returns an updated AgentLog (immutable pattern matching addLogEntry). Each wraps the call in try/catch, logging success or failure.

1. `registerIdentityAction(agent, log)`:
   - If `agent.erc8004_token_id` is truthy, log skip and return
   - Otherwise: call `buildAgentCard(agent)`, then `uploadToFilecoin()` with the card JSON, then `registerAgent()` from erc8004.ts with the Filecoin URL
   - Update agent row in DB: set erc8004_token_id
   - Log action "register_identity" with details including tokenId and txHash

2. `createBountyAction(agent, scenario, log)`:
   - Check if agent already created a demo bounty (query bounties where creator_id = agent.id AND title matches scenario.bountyToCreate.title)
   - If exists, log skip
   - Otherwise: INSERT into bounties table with crypto.randomUUID(), status "open"
   - Log action "create_bounty" with bountyId

3. `discoverAndClaimBounty(agent, log)` -> returns `{ log, bountyId: string | null }`:
   - Query `bounties WHERE status = 'open' AND required_service_type = ? AND creator_id != ? LIMIT 1` using agent.service_type
   - If none found, log "discover_bounty" with found: false, return null
   - Log discovery, then UPDATE bounty status to "claimed", claimed_by = agent.id
   - Log "claim_bounty" with bountyId
   - Return the bountyId

4. `createPostAction(agent, content, log)` -> returns `{ log, postId: string }`:
   - INSERT into posts (id, agent_id, content, media_type='text') using crypto.randomUUID()
   - Log "create_post" with postId and content preview (first 80 chars)

5. `mintPostNFTAction(agent, postId, log)`:
   - Call `mintPostNFT({ agentId: agent.id, postId })` from chain/nft.ts
   - If agent has no nft_collection_address, the mint function auto-deploys (existing behavior)
   - Log "mint_nft" with txHash and tokenId from result
   - On failure, log and continue (NFT minting is non-critical for demo flow)

6. `completeBountyAction(agent, bountyId, log)`:
   - Query bounty to verify status is "claimed" and claimed_by matches agent.id
   - If bounty has reward_amount and it is not "0", call `transferUsdc()` to agent.wallet_address
   - UPDATE bounty: status="completed", tx_hash, deliverable_url="Autonomous agent delivery", completed_at
   - Log "complete_bounty" with bountyId and txHash (if payment made)

7. `uploadLogAction(log)` -> returns `{ log, pieceCid: string | null }`:
   - Call `uploadToFilecoin()` with the log object serialized as JSON
   - Log "upload_log" with pieceCid
   - On failure, log failure and return null for pieceCid

Import `getDb` from `@/lib/db`. Import chain modules directly (not HTTP). Use `import type { Agent } from '@/lib/types'` and `import { AgentLog, addLogEntry } from '@/lib/agent-log'`.

IMPORTANT: Read the actual function signatures from the chain module source files before writing imports. The research code examples may not exactly match the real signatures. For example, check what parameters `uploadToFilecoin`, `registerAgent`, `mintPostNFT`, and `transferUsdc` actually accept.
