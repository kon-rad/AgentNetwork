  <action>
    Read node_modules/next/dist/docs/01-app/03-api-reference/03-file-conventions/route.md before writing. Check the context parameter signature for dynamic routes.

    Create src/app/api/agents/[id]/chat/route.ts with two exports:

    **GET** — returns message history:
    - Call requireOwnership(agentId); if Response, return it
    - Query supabaseAdmin.from('chat_messages').select('*').eq('agent_id', agentId).order('created_at', { ascending: true })
    - Return Response.json(data ?? [])

    **POST** — sends a message:
    - Call requireOwnership(agentId); if Response, return it
    - Parse body: { content: string }
    - Validate content is a non-empty string (max 4000 chars); return 400 if invalid
    - Insert user message: supabaseAdmin.from('chat_messages').insert({ agent_id: agentId, role: 'user', content })
    - Forward to NanoClaw: POST to `${process.env.NANOCLAW_URL}/message` with headers { Authorization: `Bearer ${process.env.NANOCLAW_SECRET}`, 'Content-Type': 'application/json' } and body { agentId, content, sender: 'user' }
    - If NanoClaw fetch fails, log the error but still return 200 (message is persisted; stream will show error)
    - Return Response.json({ ok: true })

    The route context parameter for dynamic routes in this version of Next.js uses the pattern:
    ```typescript
    export async function GET(
      request: Request,
      { params }: { params: Promise<{ id: string }> }
    ) {
      const { id: agentId } = await params;
      ...
    }
    ```
    Use this pattern for both GET and POST.

    NANOCLAW_URL is the VPS base URL (e.g. https://nanoclaw.example.com). Add it to the env var list.

  <action>
    Read node_modules/next/dist/docs/01-app/02-guides/streaming.md for current SSE/ReadableStream patterns.

    Create src/app/api/agents/[id]/chat/stream/route.ts that proxies NanoClaw's SSE endpoint to the browser.

    **GET** — proxy NanoClaw SSE:
    - Call requireOwnership(agentId); if Response, return it
    - Fetch NanoClaw SSE: `fetch(`${process.env.NANOCLAW_URL}/stream/${agentId}`, { headers: { Authorization: `Bearer ${process.env.NANOCLAW_SECRET}` } })`
    - If the upstream fetch fails, return a synthetic SSE error event and close
    - Pipe the upstream ReadableStream to the browser with correct headers:
      ```typescript
      return new Response(upstreamResponse.body, {
        headers: {
          'Content-Type': 'text/event-stream',
          'Cache-Control': 'no-cache',
          'Connection': 'keep-alive',
          'X-Accel-Buffering': 'no',
        },
      });
      ```
    - Use the same dynamic params pattern as Task 1: `{ params }: { params: Promise<{ id: string }> }`

    NanoClaw SSE event format (turn-level, not token-level):
    - `data: {"type":"thinking"}` — agent started turn
    - `data: {"type":"tool_use","tool":"<name>"}` — using a tool
    - `data: {"type":"response","content":"<text>"}` — final response text
    - `data: {"type":"done"}` — turn complete

    The client will parse these events to drive the status indicator and display the assistant message.

    After receiving a `response` event, the NEXT plan's UI will insert the assistant message into Supabase. This route is pure proxy — it does NOT write to Supabase (avoids double-write race).
