  <name>Task 1: Chat page UI</name>
  <files>src/app/agent/[id]/chat/page.tsx</files>
  <action>
    Read node_modules/next/dist/docs/01-app/03-api-reference/03-file-conventions/page.md before writing.

    Create src/app/agent/[id]/chat/page.tsx as a "use client" component.

    **Imports:**
    - useEffect, useState, useRef from react
    - useParams from next/navigation
    - ChatMessage, AgentStatus from @/lib/types

    **State:**
    - messages: ChatMessage[] — loaded from GET /api/agents/[id]/chat on mount
    - input: string — controlled textarea value
    - status: AgentStatus — 'idle' | 'thinking' | 'using tool'
    - streamingContent: string — assistant message content being built during SSE stream

    **On mount (useEffect):**
    1. Fetch GET /api/agents/[id]/chat and set messages
    2. Open EventSource to /api/agents/[id]/chat/stream
       - On message event, parse JSON:
         - type 'thinking' → setStatus('thinking')
         - type 'tool_use' → setStatus('using tool')
         - type 'response' → setStreamingContent(data.content); setStatus('idle')
         - type 'done' → if streamingContent, push a synthetic ChatMessage { id: crypto.randomUUID(), agent_id: id, role: 'assistant', content: streamingContent, created_at: new Date().toISOString() } into messages; setStreamingContent(''); close EventSource; reopen after brief delay (500ms) for next turn
    3. Cleanup: close EventSource on unmount

    **Send handler:**
    - Trim input; if empty, return
    - Optimistically add user message to messages state
    - Clear input
    - POST to /api/agents/[id]/chat with { content }
    - On error, show error in console (do not remove optimistic message — it persists via Supabase)

    **Keyboard handler for textarea:**
    - onKeyDown: if key === 'Enter' && !shiftKey → preventDefault(); call send handler
    - Shift+Enter: default browser behavior (newline)

    **Layout (cyberpunk theme, matching existing pages):**
    - Outer: max-w-3xl mx-auto px-4 py-6
    - Header: glass-card p-4 mb-4 with agent name (fetch agent name from /api/agents/[id]) and status indicator chip
    - Status indicator chip: small dot + text, colors — idle: cyan, thinking: yellow/amber, 'using tool': purple. Use inline Tailwind classes: e.g. `text-cyan-400` for idle, `text-yellow-400` for thinking, `text-purple-400` for using tool.
    - Message list: scrollable div (overflow-y-auto, max-h set to fill viewport minus header/input), messages rendered as bubbles
      - User messages: right-aligned, bg-cyan-500/10 border border-cyan-500/30 text-white
      - Assistant messages: left-aligned, glass-card (use existing CSS class) text-gray-200
      - If streamingContent is non-empty, show a streaming assistant bubble at the bottom with a blinking cursor (animate-pulse on a span)
    - Input area: glass-card p-3 sticky bottom-0 mt-4
      - Textarea (rows=3, resize-none) with placeholder "Message your agent... (Enter to send, Shift+Enter for newline)"
      - Send button: cyberpunk-button or btn-primary (use existing CSS class pattern from agent profile page)
    - Auto-scroll to bottom when messages changes: useRef on the bottom of the message list, scrollIntoView({ behavior: 'smooth' })

    Do NOT fetch agent name in a separate state — reuse the existing /api/agents/[id] pattern (one fetch, set agentName from data.display_name).

    Handle 401/403 responses from the GET history call: show a glass-card with "Access denied — you do not own this agent." message instead of the chat UI.
