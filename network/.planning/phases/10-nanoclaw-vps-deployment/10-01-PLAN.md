<task type="auto">
  <name>Task 2: Write and run VPS provisioning script</name>
  <files>agent-server/scripts/provision-vps.sh</files>
  <action>
    Create `agent-server/scripts/provision-vps.sh` — an idempotent bootstrap script to be run once on the VPS as root via `ssh root@VPS_IP bash -s < provision-vps.sh`.

    The script must:
    1. Update apt and install prerequisites (curl, gnupg, ca-certificates, lsb-release, git, wireguard-tools)
    2. Add Docker's official apt repo and install: docker-ce, docker-ce-cli, containerd.io, docker-buildx-plugin, docker-compose-plugin (from https://download.docker.com/linux/ubuntu)
    3. Install Node.js 20 LTS via NodeSource (curl -fsSL https://deb.nodesource.com/setup_20.x | bash - && apt install -y nodejs)
    4. Install Caddy via official cloudsmith apt repo (see research Pattern 3 for commands)
    5. Enable and start Docker service: `systemctl enable docker && systemctl start docker`
    6. Enable and start Caddy service: `systemctl enable caddy && systemctl start caddy`
    7. Create deploy user if not exists: `useradd -m -s /bin/bash deploy` (skip if already exists)
    8. Add deploy user to docker group: `usermod -aG docker deploy`
    9. Create /opt/agent-server owned by deploy: `mkdir -p /opt/agent-server && chown deploy:deploy /opt/agent-server`
    10. Print confirmation: `docker --version`, `node --version`, `caddy version`

    Use `set -euo pipefail` at the top. Make each step idempotent (check before install).

    After writing the script, run it against the VPS using the IP and username provided in Task 1:
    ```
    ssh -o StrictHostKeyChecking=no VPS_USER@VPS_IP 'bash -s' < agent-server/scripts/provision-vps.sh
    ```

    Then verify as the deploy user:
    ```
    ssh deploy@VPS_IP 'docker ps && node --version && caddy version'
    ```
  </action>
  <verify>

    - `docker ps` shows empty table (no error)
    - `node --version` shows v20.x.x
    - `caddy version` shows v2.x.x
  </verify>
  <done>Deploy user on VPS can run Docker commands and Node.js without sudo. Caddy service is running. VPS is ready for NanoClaw deployment.</done>
