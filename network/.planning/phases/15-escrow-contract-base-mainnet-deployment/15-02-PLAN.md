<objective>
Create API routes at /api/escrow that expose the full escrow job lifecycle: create (verify client tx), read status, release, dispute, resolve (treasury), and refund (treasury).

Purpose: Provide HTTP endpoints for the frontend to interact with the on-chain escrow contract. Client-signed operations (create, release, dispute) are verified server-side after the client signs in their wallet. Treasury operations (resolve, refund) are signed server-side with TREASURY_PRIVATE_KEY.

Output: 5 API route files covering the complete escrow job lifecycle.
</objective>

Create `src/app/api/escrow/route.ts` following the subscriptions/route.ts pattern:

**POST /api/escrow** — Verify on-chain job creation
- `requireAuth()` for session check
- Accept JSON body: `{ tx_hash: string }`
- Call `publicClient.getTransactionReceipt({ hash: tx_hash })` on Base mainnet
- Verify receipt.status === 'success'
- Parse JobCreated event from receipt logs using `decodeEventLog` with the escrow ABI (import from escrow.ts)
- Extract jobId, client, agent, amount from the decoded event
- Verify `receipt.from.toLowerCase() === session.address.toLowerCase()` (sender matches authenticated wallet)
- Return `{ jobId, client, agent, amount, status: 'Active', txHash: tx_hash, basescanUrl: 'https://basescan.org/tx/...' }`
- Use `createPublicClient({ chain: base, transport: http() })` at module level (same pattern as subscriptions/route.ts)

**GET /api/escrow?jobId=N** — Read job status from on-chain
- `requireAuth()` for session check
- Parse `jobId` from search params
- Call `getJob(BigInt(jobId))` from escrow.ts
- Return the job data with status mapped from enum (0=Active, 1=Released, 2=Disputed, 3=Resolved, 4=Refunded)

Create 4 route files under `src/app/api/escrow/[jobId]/`:

**POST /api/escrow/[jobId]/release** — Verify client released funds
- `requireAuth()` for session check
- Accept JSON body: `{ tx_hash: string }`
- Get receipt, verify success, parse JobReleased event
- Verify sender matches session wallet
- Call `getJob(BigInt(jobId))` to confirm status is now Released (1)
- Return `{ jobId, status: 'Released', txHash }`

**POST /api/escrow/[jobId]/dispute** — Verify dispute raised
- `requireAuth()` for session check
- Accept JSON body: `{ tx_hash: string }`
- Get receipt, verify success, parse JobDisputed event
- Verify sender matches session wallet
- Call `getJob(BigInt(jobId))` to confirm status is now Disputed (2)
- Return `{ jobId, status: 'Disputed', txHash }`

**POST /api/escrow/[jobId]/resolve** — Treasury resolves dispute (server-side signing)
- `requireAuth()` for session check
- Verify `session.address.toLowerCase() === process.env.TREASURY_ADDRESS?.toLowerCase()` — only treasury wallet can resolve
- Accept JSON body: `{ toAgentUsdc: string, toClientUsdc: string, toTreasuryUsdc: string }`
- Call `resolveDispute(BigInt(jobId), toAgentUsdc, toClientUsdc, toTreasuryUsdc)` from escrow.ts (server-side signing with TREASURY_PRIVATE_KEY)
- Return `{ jobId, status: 'Resolved', txHash }`

**POST /api/escrow/[jobId]/refund** — Treasury refunds job (server-side signing)
- `requireAuth()` for session check
- Verify `session.address.toLowerCase() === process.env.TREASURY_ADDRESS?.toLowerCase()` — only treasury
- Call `refundJob(BigInt(jobId))` from escrow.ts (server-side signing)
- Return `{ jobId, status: 'Refunded', txHash }`

All routes extract `jobId` from the route params. Use this Next.js pattern:
```typescript
export async function POST(
  req: NextRequest,
  { params }: { params: Promise<{ jobId: string }> }
) {
  const { jobId } = await params
  // ...
}
```

Error handling pattern (consistent across all routes):
- 400 for missing/invalid params
- 401 from requireAuth()
- 403 for treasury-only routes when caller is not treasury
- 422 for on-chain verification failures
- 500 for server errors
