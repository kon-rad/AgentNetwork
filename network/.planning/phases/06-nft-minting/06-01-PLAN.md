<task type="auto">
  <name>Task 2: Create deploy-collection and mint-nft API routes</name>
  <files>src/app/api/chain/deploy-collection/route.ts, src/app/api/chain/mint-nft/route.ts</files>
  <action>
    1. Create `src/app/api/chain/deploy-collection/route.ts`:
       - POST handler accepting `{ agentId: string }`
       - Look up agent from DB; return 404 if not found
       - If agent already has `nft_collection_address`, return 200 with existing address (idempotent, same pattern as erc8004 registration)
       - Call `deployCollection(agent.display_name)` from chain/nft.ts
       - Update DB: `UPDATE agents SET nft_collection_address = ? WHERE id = ?`
       - Return 201 with `{ contractAddress, txHash }`
       - Catch SDK errors and return 502 (upstream service failure, same pattern as phase 02-02)

    2. Create `src/app/api/chain/mint-nft/route.ts`:
       - POST handler accepting `{ postId: string }`
       - Look up post (with agent join) from DB; return 404 if not found
       - If post already has `nft_contract`, return 200 with existing NFT data (idempotent)
       - If agent has no `nft_collection_address`, deploy collection first by calling deployCollection and updating agent record
       - Build ERC-721 metadata JSON:
         ```
         { name: "Post by {agent.display_name}", description: post.content,
           external_url: "/agent/{agent.id}",
           attributes: [{ trait_type: "Agent", value: agent.display_name },
                        { trait_type: "Service Type", value: agent.service_type }] }
         ```
       - Upload metadata to Filecoin: `uploadToFilecoin(metadata, "nft_{postId}.json")`
       - Use `retrievalUrl` (NOT raw pieceCid) as tokenURI for the mint
       - Call `mintPostNFT({ collectionAddress, toAddress: agent.wallet_address, tokenUri: retrievalUrl })`
       - Update DB: `UPDATE posts SET nft_contract = ?, nft_token_id = ?, filecoin_cid = ? WHERE id = ?`
       - Also record in filecoin_uploads table with upload_type = 'nft_metadata'
       - Return 201 with `{ nftContract, tokenId, txHash, filecoinCid }`
       - Catch errors: Filecoin failures = 502, SDK failures = 502, validation = 400

    Use `getDb()` from `@/lib/db` for all database operations. Use `crypto.randomUUID()` for filecoin_uploads record IDs (established pattern from phase 02-02).
  </action>
  <verify>
    - `npx tsc --noEmit` passes
    - `pnpm build` succeeds
    - Both route files exist and export POST handler
  </verify>

    - POST /api/chain/mint-nft mints post as NFT with Filecoin metadata (idempotent)
    - DB updated with collection address, NFT contract, token ID, and Filecoin CID
  </done>
