<task type="auto">
  <name>Task 1: Create upload API route with DB tracking</name>
  <files>
    src/app/api/chain/upload/route.ts
  </files>
  <action>
Create `POST /api/chain/upload` route at `src/app/api/chain/upload/route.ts`:

1. Import `uploadToFilecoin` from `@/lib/chain/filecoin`
2. Import `getDb` from `@/lib/db`
3. Import `v4 as uuid` from `uuid` (already in project deps — check, else use crypto.randomUUID())
4. Import `FilecoinUploadType` from `@/types/filecoin`

5. Export async function POST(req: Request):
   - Parse JSON body: `{ type, agentId, data, name }`
   - Validate `type` is one of `['agent_card', 'agent_log', 'nft_metadata']` — return 400 if invalid
   - Validate `agentId` is a non-empty string — return 400 if missing
   - Validate `data` is a non-null object — return 400 if missing
   - If `name` is not provided, default to `${type}_${agentId}.json`

   - For `agent_card` type: check if this agent already has an agent_card upload in filecoin_uploads.
     If exists, still allow re-upload (agent cards can be updated) but log that it's replacing a previous version.

   - Call `uploadToFilecoin(data, name)` — this may take seconds to minutes (waits for onPiecesConfirmed)
   - On success, insert into filecoin_uploads table:
     ```
     INSERT INTO filecoin_uploads (id, agent_id, upload_type, piece_cid, retrieval_url, name)
     VALUES (?, ?, ?, ?, ?, ?)
     ```
   - Return 201 with JSON: `{ pieceCid, retrievalUrl, uploadType, agentId, id }`

   - Wrap entire handler in try/catch:
     - On SDK errors (upload failure, payment errors), return 502 with `{ error: 'Filecoin upload failed', details: err.message }`
     - On validation errors, return 400
     - On unexpected errors, return 500

   - Do NOT set `export const runtime = 'edge'` — Synapse SDK requires Node.js runtime
  </action>
  <verify>
    - `npx tsc --noEmit` passes
    - `grep -q "uploadToFilecoin" src/app/api/chain/upload/route.ts` confirms wiring
    - `pnpm build` succeeds
  </verify>
  <done>
    - POST /api/chain/upload accepts { type, agentId, data, name } and returns { pieceCid, retrievalUrl }
    - Upload result is persisted to filecoin_uploads table in SQLite
    - Invalid type/agentId returns 400; upload failure returns 502
  </done>

   - Optional query param `type` to filter by upload_type: `?type=agent_card`
   - Return 200 with JSON array of FilecoinUploadRecord objects
   - If agent has no uploads, return empty array (not 404)

Both routes follow existing project patterns from ARCHITECTURE.md:
