<task type="auto">
  <name>Task 2: Create Filecoin adapter and add DB schema</name>
  <files>

  <action>
1. Create `src/lib/chain/filecoin.ts` — server-only Synapse adapter:
   - Add `import 'server-only'` as first line (prevents client-side import)
   - Import `Synapse` from `@filoz/synapse-sdk`
   - Import `getChain` from `@filoz/synapse-core/chains`
   - Import `createWalletClient`, `http`, `type Hex` from `viem`
   - Import `privateKeyToAccount` from `viem/accounts`
   - Import `FilecoinUploadResult` from `@/types/filecoin`

   - Determine chain ID from env: `FILECOIN_NETWORK === 'mainnet' ? 314 : 314159`
   - Implement singleton `getSynapse()`: creates viem wallet client with private key, returns `new Synapse({ client })`
   - Cache the Synapse instance in module-level variable (same pattern as db.ts)

   - Export `uploadToFilecoin(data: object, name: string): Promise<FilecoinUploadResult>`:
     - Serialize data to JSON string with 2-space indent
     - Encode to Uint8Array via TextEncoder
     - Create ReadableStream wrapping the bytes
     - Call `synapse.storage.createContext({ withCDN: true })`
     - Upload stream with `context.upload(stream, { pieceMetadata: { name } })`
     - Wait for `onPiecesConfirmed` callback to get PieceCID (NOT onStored — per research, onPiecesConfirmed means data is retrievable)
     - Return `{ pieceCid, retrievalUrl: 'https://cdn.filecoin.cloud/${pieceCid}' }`
     - Wrap in Promise to handle async callback pattern

   - Export `downloadFromFilecoin(pieceCid: string): Promise<object>`:
     - Call `synapse.download(pieceCid)`
     - Decode Uint8Array to string via TextDecoder
     - Parse JSON and return

   - IMPORTANT: Do NOT use `export const runtime = 'edge'` — SDK needs Node.js runtime
   - IMPORTANT: Use `getChain()` with Filecoin chain IDs (314/314159), NOT Base chain IDs (8453)

2. Update `src/lib/db.ts` — add filecoin_uploads table to `initSchema()`:
   ```sql
   CREATE TABLE IF NOT EXISTS filecoin_uploads (
     id TEXT PRIMARY KEY,
     agent_id TEXT NOT NULL REFERENCES agents(id),
     upload_type TEXT NOT NULL,
     piece_cid TEXT NOT NULL,
     retrieval_url TEXT NOT NULL,
     name TEXT NOT NULL,
     created_at TEXT DEFAULT (datetime('now'))
   );
   CREATE INDEX IF NOT EXISTS idx_filecoin_uploads_agent ON filecoin_uploads(agent_id);
   CREATE INDEX IF NOT EXISTS idx_filecoin_uploads_type ON filecoin_uploads(agent_id, upload_type);
   ```
   This table tracks all Filecoin uploads per agent, queryable by agent_id and upload_type.
   Add these CREATE statements inside the existing `db.exec()` template literal, after the existing index statements.
