<task type="auto">
  <name>Task 2: Create deploy-token and deploy-all-tokens API routes</name>
  <files>src/app/api/chain/deploy-token/route.ts, src/app/api/chain/deploy-all-tokens/route.ts</files>
  <action>
**POST /api/chain/deploy-token/route.ts** — Deploy a single agent token:
- Accept JSON body: `{ agentId: string }`
- Look up agent from DB: `SELECT id, display_name, token_symbol, token_address FROM agents WHERE id = ?`
- Return 404 if agent not found
- Return 400 if agent has no `token_symbol`
- Return 409 if agent already has `token_address` (already deployed — include existing address in response)
- Call `deployAgentToken(`${display_name} Token`, token_symbol)` from clanker.ts
- On success: `UPDATE agents SET token_address = ?, updated_at = datetime('now') WHERE id = ?`
- Return 200 with `{ agentId, symbol, tokenAddress, txHash? }`
- On Clanker SDK failure: return 502 (upstream failure, same pattern as Filecoin routes per decision [02-02])

**POST /api/chain/deploy-all-tokens/route.ts** — Deploy tokens for all agents without one:
- No request body needed
- Query: `SELECT id, display_name, token_symbol FROM agents WHERE token_address IS NULL AND token_symbol IS NOT NULL`
- If no agents need tokens: return 200 with `{ deployed: [], message: "All agents already have tokens" }`
- Loop sequentially (NOT parallel — nonce conflicts per research pitfall 3)
- For each agent: call deployAgentToken, update DB on success, catch errors per agent
- Return 200 with results array: `{ deployed: [{ agentId, symbol, tokenAddress, success }], failed: [{ agentId, symbol, error }] }`
- Use try/catch per agent so one failure does not abort the batch

Both routes: import `getDb` from `@/lib/db` and `deployAgentToken` from `@/lib/chain/clanker`.
  </action>

  </verify>
