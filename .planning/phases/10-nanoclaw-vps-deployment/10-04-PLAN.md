    **Create agent-server/Dockerfile (NanoClaw host process):**
    ```dockerfile
    FROM node:20-slim

    WORKDIR /app

    # Install Docker CLI (needed to spawn per-turn containers via Docker socket)
    RUN apt-get update && apt-get install -y docker.io && rm -rf /var/lib/apt/lists/*

    COPY package*.json ./
    RUN npm ci --only=production

    COPY dist/ ./dist/

    # NanoClaw data directory (SQLite + IPC + group workspaces)
    RUN mkdir -p /app/data

    EXPOSE 3000

    CMD ["node", "dist/index.js"]
    ```

    **Create agent-server/container/Dockerfile (per-turn agent container — nanoclaw-agent:latest):**
    ```dockerfile
    FROM node:22-slim

    WORKDIR /agent

    # Install Claude Agent SDK (Anthropic SDK + agent runner)
    # Use the npm package that NanoClaw's container-runner.ts expects
    RUN npm install -g @anthropic-ai/claude-code 2>/dev/null || \
        npm install @anthropic-ai/sdk

    # Per-turn containers receive task via IPC volume mount
    # The entry point is set by NanoClaw's container-runner.ts at spawn time
    CMD ["node", "-e", "console.log('nanoclaw-agent ready')"]
    ```
    Note: Read agent-server/src/container-runner.ts from the upstream copy to confirm the exact base image and entry point expected. Adjust if different.

    **Create agent-server/docker-compose.yml:**

    Read TRANSPORT.md first to determine which transport to use.

    If TRANSPORT.md says HTTPS+Caddy:
    ```yaml
    services:
      nanoclaw:
        build:
          context: .
          dockerfile: Dockerfile
        restart: always
        environment:
          - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
          - WEBAPP_PORT=3000
          - WEBAPP_SHARED_SECRET=${WEBAPP_SHARED_SECRET}
          - SUPABASE_URL=${SUPABASE_URL}
          - SUPABASE_SERVICE_ROLE_KEY=${SUPABASE_SERVICE_ROLE_KEY}
        volumes:
          - ./data:/app/data
          - /var/run/docker.sock:/var/run/docker.sock
        networks:
          - internal
        # Port 3000 NOT exposed publicly — Caddy accesses via internal network
        # Credential proxy is exposed on 3001 so per-turn agent containers can reach it (NC-04)
        ports:
          - "127.0.0.1:3001:3001"

      caddy:
        image: caddy:2-alpine
        restart: always
        ports:
          - "80:80"
          - "443:443"
        volumes:
          - ./Caddyfile:/etc/caddy/Caddyfile
          - caddy_data:/data
          - caddy_config:/config
        networks:
          - internal

    networks:
      internal:
        driver: bridge

    volumes:
      caddy_data:
      caddy_config:
    ```

    **CRITICAL (NC-04):** After writing docker-compose.yml, confirm that NanoClaw's container-runner.ts passes `ANTHROPIC_BASE_URL=http://host.docker.internal:3001` (or the internal network equivalent) to each spawned agent container. If container-runner.ts uses a different env var name, match it. The credential proxy must be the sole path for API calls — per-turn containers must NOT receive `ANTHROPIC_API_KEY` directly.

    Check how NanoClaw's container-runner.ts spawns containers (from the upstream source copied in 10-03). If it does not already inject `ANTHROPIC_BASE_URL`, add it to the `docker run` env args in container-runner.ts. Document this in the summary.

    If TRANSPORT.md says WireGuard: use same compose but without caddy service, and expose port 3000 only on WireGuard interface (10.0.0.1). Keep credential proxy port binding as above.

    **Create agent-server/Caddyfile (only needed for HTTPS+Caddy transport):**

    Read TRANSPORT.md to get the domain name. The user will need to create a DNS A record pointing their domain to VPS_IP before Caddy can issue the Let's Encrypt certificate.

    ```
    nanoclaw.YOURDOMAIN.com {
        # SSE endpoint needs long timeout (agent turns can take minutes)
        handle /stream/* {
            reverse_proxy nanoclaw:3000 {
                transport http {
                    response_header_timeout 300s
                    read_timeout 300s
                    write_timeout 300s
                }
            }
        }

        # All other endpoints (including /message, /register-group)
        reverse_proxy nanoclaw:3000

        tls {
            protocols tls1.3
        }
    }
    ```

    Replace `nanoclaw.YOURDOMAIN.com` with the actual domain from TRANSPORT.md (or if no domain is set yet, use a placeholder and note that the DNS A record must be created before `docker compose up`).
