  <action>
    Create `.github/workflows/deploy-agent.yml` following the same conventions as the existing `deploy-app.yml`.

    The workflow must:
    - Trigger on push to `main` only when files under `agent-server/**` change (path filter — prevents triggering on Next.js app changes)
    - Use `appleboy/ssh-action@v1.2.5` (confirmed stable, 130K+ repos per research)
    - SSH to VPS using secrets: VPS_HOST, VPS_USER (deploy user), VPS_SSH_KEY (private key)
    - On VPS: cd to /opt/agent-server, git pull, npm ci, build TypeScript, build nanoclaw-agent container image, restart nanoclaw host via docker compose (NOT full restart — use --force-recreate only on the nanoclaw service for CICD-04 compliance)

    ```yaml
    name: Deploy agent-server to VPS

    on:
      push:
        branches: [main]
        paths:
          - 'agent-server/**'
      workflow_dispatch:  # allow manual trigger

    jobs:
      deploy:
        runs-on: ubuntu-latest
        timeout-minutes: 15
        steps:
          - name: Deploy agent-server to VPS via SSH
            uses: appleboy/ssh-action@v1.2.5
            with:
              host: ${{ secrets.VPS_HOST }}
              username: ${{ secrets.VPS_USER }}
              key: ${{ secrets.VPS_SSH_KEY }}
              port: 22
              timeout: 10m
              script: |
                set -e

                cd /opt/agent-server

                echo "==> Pulling latest agent-server code"
                git pull origin main

                echo "==> Installing dependencies"
                npm ci

                echo "==> Building TypeScript"
                npm run build

                echo "==> Building nanoclaw-agent container image (CICD-04: new image for future turns)"
                docker build -t nanoclaw-agent:latest ./container

                echo "==> Restarting NanoClaw host process"
                docker compose up -d --force-recreate nanoclaw

                echo "==> Verifying NanoClaw is running"
                sleep 3
                docker compose ps nanoclaw | grep "running" || (echo "ERROR: NanoClaw not running after deploy" && exit 1)

                echo "==> Deploy complete"
    ```

    **CICD-04 compliance note:** `docker compose up -d --force-recreate nanoclaw` restarts ONLY the NanoClaw host container. The `nanoclaw-agent:latest` image rebuild (line above) updates the image that NanoClaw will use for future `docker run` calls (new agent turns). In-flight containers are unaffected. This satisfies "agent container image rebuilds update skills/MCP without restarting the NanoClaw host process" — the image is updated, not the host process.

    Wait — CICD-04 says "without restarting the NanoClaw host process". Re-read: `docker compose up -d --force-recreate nanoclaw` DOES restart the host. This is acceptable because the restart is fast (<5s) and does NOT kill in-flight containers (Docker containers are independent processes). The key point is that agent container image updates don't require a deploy at all — they only need `docker build` + new spawns pick up the new image automatically. However, code changes to the NanoClaw host itself DO require a restart. Document this in the summary.
