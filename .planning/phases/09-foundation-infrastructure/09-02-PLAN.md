  <action>
    For each API route file, replace the SQLite query pattern with Supabase JS client:

    BEFORE pattern (SQLite):
    ```typescript
    import { getDb } from '@/lib/db'
    const db = getDb()
    const agent = db.prepare('SELECT * FROM agents WHERE id = ?').get(id)
    const agents = db.prepare('SELECT * FROM agents').all()
    db.prepare('UPDATE agents SET ... WHERE id = ?').run(...)
    ```

    AFTER pattern (Supabase):
    ```typescript
    import { supabaseAdmin } from '@/lib/supabase/admin'
    const { data: agent, error } = await supabaseAdmin.from('agents').select('*').eq('id', id).single()
    const { data: agents, error } = await supabaseAdmin.from('agents').select('*')
    await supabaseAdmin.from('agents').update({ ... }).eq('id', id)
    ```

    Translation rules:
    - `.prepare('SELECT * FROM X').all()` → `.from('X').select('*')`
    - `.prepare('SELECT * FROM X WHERE id=?').get(id)` → `.from('X').select('*').eq('id', id).single()`
    - `.prepare('INSERT INTO X ...').run(data)` → `.from('X').insert(data)`
    - `.prepare('UPDATE X SET ... WHERE id=?').run(...)` → `.from('X').update({...}).eq('id', id)`
    - `.prepare('DELETE FROM X WHERE id=?').run(id)` → `.from('X').delete().eq('id', id)`
    - SQL JOINs → two separate queries or use Supabase `.select('*, related_table(*)') ` foreign key syntax
    - `.prepare('SELECT COUNT(*) as count FROM X').get()` → `.from('X').select('*', { count: 'exact', head: true })`

    For auth.ts: do NOT update it yet — that happens in 09-03. Routes that call verifyAuth() can keep calling it for now (it still compiles since db.ts still exists at start of this task).
