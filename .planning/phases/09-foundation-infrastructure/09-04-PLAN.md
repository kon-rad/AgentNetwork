  <name>Task 2: Wire GitHub Actions + configure Railway monorepo settings</name>
  <what-built>
    GitHub Actions workflow file at .github/workflows/deploy-app.yml that triggers Railway deploy when app/** changes are pushed to main.

    Also requires manual Railway dashboard configuration (Root Directory = app) which Claude cannot do via CLI.
  </what-built>
  <action>
    Claude creates the workflow file first:

    .github/workflows/deploy-app.yml:
    ```yaml
    name: Deploy app to Railway

    on:
      push:
        branches: [main]
        paths:
          - 'app/**'
          - 'pnpm-workspace.yaml'

    jobs:
      deploy:
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4

          - name: Deploy to Railway
            uses: railway/deploy-action@v1
            with:
              service: ${{ secrets.RAILWAY_SERVICE_ID }}
              token: ${{ secrets.RAILWAY_TOKEN }}
    ```

    Then the user must do two manual steps before this can be tested:

    Step A — Add GitHub Secrets:
    1. Go to GitHub repo → Settings → Secrets and variables → Actions → New repository secret
    2. Add RAILWAY_TOKEN: Railway Dashboard → Account Settings → Tokens → Create token
    3. Add RAILWAY_SERVICE_ID: Railway project → service → Settings → Service ID (copy from URL or settings)

    Step B — Set Railway Root Directory:
    1. Railway Dashboard → your project → Next.js service → Settings → Source
    2. Set "Root Directory" to: `app`
    3. Save → Railway will redeploy using app/ as the root

    After both steps, push a trivial change to app/ and verify the GitHub Actions run deploys successfully.
  </action>
  <how-to-verify>
    1. Check .github/workflows/deploy-app.yml exists in repo
    2. Add RAILWAY_TOKEN and RAILWAY_SERVICE_ID to GitHub Secrets (see steps above)
    3. Set Railway Root Directory to `app` (see steps above)
    4. Make a trivial change to app/src/app/page.tsx (e.g., add a comment), commit, push to main
    5. GitHub → Actions tab → "Deploy app to Railway" workflow should trigger and succeed (green checkmark)
    6. Railway dashboard → Deployments shows a new deployment triggered
  </how-to-verify>
