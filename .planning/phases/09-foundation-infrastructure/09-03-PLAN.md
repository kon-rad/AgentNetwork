<objective>
Implement SIWE (Sign-In With Ethereum) authentication with iron-session cookie sessions. Replace the custom EIP-191 per-request auth with a proper sign-in flow: nonce → wallet sign → session cookie. Add owner_wallet field type and ownership guard helpers for auth and ownership checks.

Purpose: The platform needs persistent user sessions for chat, observability, and agent management. Per-request signing is not viable for v2.0 features. SIWE + iron-session is the standard minimal pattern for wallet-based auth in Next.js.

Note on OWN-03 (RLS policies): Because iron-session is application-layer auth and not Supabase Auth, standard Supabase RLS policies cannot reference the session directly. Ownership enforcement in Phase 9 is implemented at the application layer via requireOwnership() middleware that checks session.address === agent.owner_wallet. True Supabase RLS policies using JWT claims will be added in Phase 11 when a Supabase Auth integration is introduced. This is documented in the agents table comment added in 09-01.

Output: Full sign-in/sign-out flow, session persistence, requireAuth() and requireOwnership() guards, navbar Sign In button.
</objective>

    Extend src/lib/auth/guard.ts with a requireOwnership() helper that implements OWN-02 and OWN-03 (application-layer ownership enforcement).

    Add to src/lib/auth/guard.ts:

    ```typescript
    import { supabaseAdmin } from '@/lib/supabase/admin'

    /**
     * Ownership guard — OWN-02 / OWN-03
     *
     * Checks that the authenticated session belongs to the agent owner.
     * Application-layer enforcement (iron-session is not Supabase Auth, so RLS
     * policies cannot reference this session directly). True RLS with JWT claims
     * will be added in Phase 11. Until then, this function is the ownership boundary.
     *
     * Usage in route handler:
     *   const ownerOrError = await requireOwnership(agentId)
     *   if (ownerOrError instanceof Response) return ownerOrError
     *   // ownerOrError is the session — continue with handler
     */
    export async function requireOwnership(agentId: string): Promise<IronSessionData | Response> {
      // Step 1: require authentication
      const sessionOrError = await requireAuth()
      if (sessionOrError instanceof Response) return sessionOrError
      const session = sessionOrError

      // Step 2: look up the agent's owner_wallet
      const { data: agent, error } = await supabaseAdmin
        .from('agents')
        .select('owner_wallet')
        .eq('id', agentId)
        .single()

      if (error || !agent) {
        return Response.json({ error: 'Agent not found' }, { status: 404 })
      }

      // Step 3: if agent has no owner (legacy agent), deny management access
      // Set owner_wallet via the claim endpoint to take ownership
      if (!agent.owner_wallet) {
        return Response.json({ error: 'Agent has no owner — claim ownership first' }, { status: 403 })
      }

      // Step 4: enforce ownership
      if (session.address?.toLowerCase() !== agent.owner_wallet.toLowerCase()) {
        return Response.json({ error: 'Forbidden — you are not the owner of this agent' }, { status: 403 })
      }

      return session
    }
    ```

    This function is intended for use on routes that manage agent configuration: chat, observability feeds, and management endpoints (register, service config, etc.). The routes themselves will use requireOwnership() starting in Phase 11 when those features are built. For now, the guard must exist and be exportable so Phase 11 plans can reference it without additional setup.
